Three server-side issues break per-project Brand DNA. Every piece of the Strategy page depends on these being fixed before the UI is built. Fix them in order before touching the frontend.

Server fix 1 — POST /api/brand-dna/refresh-intelligence must be project-aware
Currently this route always calls storage.getBrandDna(userId) which returns the global DNA record, ignoring the project entirely. Agence JMD and Naya are completely different brands — a global DNA summary is meaningless.
In server/routes.ts, modify the route:
tsapp.post('/api/brand-dna/refresh-intelligence', isAuthenticated, async (req: any, res) => {
  const userId = req.user.claims.sub;
  const projectId = req.body.projectId ? parseInt(req.body.projectId) : null;

  // Always use per-project DNA — never global
  const dna = projectId
    ? await storage.getBrandDnaForProject(userId, projectId)
    : await storage.getBrandDna(userId);

  if (!dna) return res.status(404).json({ message: "Brand DNA not found for this project" });

  // ... existing prompt construction using dna fields ...

  // Save back to the correct record
  const updated = projectId
    ? await storage.upsertBrandDnaForProject(userId, projectId, {
        nayaIntelligenceSummary: summary,
        lastStrategyRefreshAt: new Date(),
      })
    : await storage.upsertBrandDna({ ...dnaFields, nayaIntelligenceSummary: summary, lastStrategyRefreshAt: new Date(), userId });

  res.json({ summary, updatedAt: updated.lastStrategyRefreshAt, projectId });
});
The prompt construction remains identical — just replace brandDna with dna throughout.
