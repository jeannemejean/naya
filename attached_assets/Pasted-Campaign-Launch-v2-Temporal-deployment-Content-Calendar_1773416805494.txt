Campaign Launch v2 — Temporal deployment + Content Calendar injection
The problem
When a campaign is launched today, all tasks land on todayStr (today's date) regardless of their phase, and the content pieces from contentPlan are never created anywhere. The campaign already contains everything needed to deploy correctly across time: phases with durations, tasks with phase numbers, and content pieces with week labels and platforms. This step makes the launch route read all of that and distribute everything correctly across the calendar.

Core concept: date calculation from campaign structure
The launch route receives an optional startDate (YYYY-MM-DD) from the request body. If not provided, default to today. All task and content dates are computed from this anchor date using the phase structure.
Phase → date range mapping:
Parse each phase's duration field (e.g. "Weeks 1–2", "Week 3", "Months 1–2", "Month 3") to extract the start week/month offset. Use this to compute an absolute date range for each phase.
Use this helper logic (implement in the route):
tsfunction parsePhaseDateRange(duration: string, startDate: Date, campaignDurationStr: string): { start: Date; end: Date } {
  // duration examples: "Weeks 1–2", "Week 3", "Month 1", "Months 2–3", "Weeks 5–8"
  // Parse the start number from the duration string
  const weekMatch = duration.match(/[Ww]eeks?\s*(\d+)/);
  const monthMatch = duration.match(/[Mm]onths?\s*(\d+)/);
  
  const start = new Date(startDate);
  const end = new Date(startDate);
  
  if (weekMatch) {
    const weekNum = parseInt(weekMatch[1]);
    start.setDate(start.getDate() + (weekNum - 1) * 7);
    // End: look for end week number e.g. "Weeks 1-2" → end at week 2
    const endWeekMatch = duration.match(/[–\-]\s*(\d+)/);
    const endWeek = endWeekMatch ? parseInt(endWeekMatch[1]) : weekNum;
    end.setDate(startDate.getDate() + (endWeek * 7) - 1);
  } else if (monthMatch) {
    const monthNum = parseInt(monthMatch[1]);
    start.setMonth(start.getMonth() + (monthNum - 1));
    const endMonthMatch = duration.match(/[–\-]\s*(\d+)/);
    const endMonth = endMonthMatch ? parseInt(endMonthMatch[1]) : monthNum;
    end.setMonth(startDate.getMonth() + endMonth);
    end.setDate(end.getDate() - 1);
  }
  
  return { start, end };
}

function dateToStr(d: Date): string {
  return d.getFullYear() + '-' + String(d.getMonth() + 1).padStart(2, '0') + '-' + String(d.getDate()).padStart(2, '0');
}

function addWeeks(d: Date, weeks: number): Date {
  const result = new Date(d);
  result.setDate(result.getDate() + weeks * 7);
  return result;
}

function addDays(d: Date, days: number): Date {
  const result = new Date(d);
  result.setDate(result.getDate() + days);
  return result;
}
Task scheduling logic:
Each task has a phase number. Map each phase number to its date range using parsePhaseDateRange. Schedule the task on the first Monday of that phase's date range — or the phase start date if it's already a weekday. Space multiple tasks within the same phase across the week: task 1 → Monday, task 2 → Wednesday, task 3 → Friday, task 4 → next Monday, etc.
Content piece scheduling logic:
Each content piece has a week field (e.g. "Week 1", "Week 3", "Month 2"). Parse this similarly — "Week N" means startDate + (N-1) * 7 days. "Month N" means startDate + (N-1) * 30 days. Schedule the scheduledFor timestamp for the Wednesday of that week at 10:00 AM (good default publication time).

Task 2 — Server: rebuild /api/campaigns/:id/launch completely
Replace the entire launch route with this logic:
tsapp.post('/api/campaigns/:id/launch', isAuthenticated, async (req: any, res) => {
  const userId = req.user.claims.sub;
  const id = parseInt(req.params.id);
  const campaign = await storage.getCampaign(id, userId);
  if (!campaign) return res.status(404).json({ message: "Campaign not found" });
  if (campaign.status === 'active') return res.status(400).json({ message: "Campaign already launched" });

  // Resolve start date — from body, campaign record, or today
  const rawStart = req.body.startDate || campaign.startDate;
  const startDate = rawStart ? new Date(rawStart) : new Date();

  const phases = (campaign.phases || []) as CampaignPhase[];
  const generatedTasks = (campaign.generatedTasks || []) as GeneratedTask[];
  const contentPlan = (campaign.contentPlan || []) as ContentPiece[];

  // ── 1. Build phase → date range map ──────────────────────────────
  const phaseMap: Record<number, { start: Date; end: Date }> = {};
  for (const phase of phases) {
    phaseMap[phase.number] = parsePhaseDateRange(phase.duration, startDate, campaign.duration || '1_month');
  }

  // Fallback for tasks with no phase: spread from startDate
  const fallbackPhaseStart = startDate;

  // ── 2. Create tasks with correct scheduledDate ────────────────────
  // Group tasks by phase, then space them within the phase week
  const tasksByPhase: Record<number, typeof generatedTasks> = {};
  for (const t of generatedTasks) {
    const p = t.phase || 1;
    if (!tasksByPhase[p]) tasksByPhase[p] = [];
    tasksByPhase[p].push(t);
  }

  let tasksCreated = 0;
  for (const [phaseNumStr, phaseTasks] of Object.entries(tasksByPhase)) {
    const phaseNum = parseInt(phaseNumStr);
    const phaseRange = phaseMap[phaseNum] || { start: fallbackPhaseStart, end: addWeeks(fallbackPhaseStart, 1) };
    
    // Distribute tasks across the phase: Monday, Wednesday, Friday, next Monday...
    const WEEKDAY_OFFSETS = [0, 2, 4, 7, 9, 11, 14]; // Mon, Wed, Fri, Mon, Wed, Fri, Mon
    
    phaseTasks.forEach((t, idx) => {
      const dayOffset = WEEKDAY_OFFSETS[idx % WEEKDAY_OFFSETS.length] + Math.floor(idx / WEEKDAY_OFFSETS.length) * 14;
      const taskDate = addDays(phaseRange.start, dayOffset);
      // Don't schedule past the phase end
      const clampedDate = taskDate <= phaseRange.end ? taskDate : phaseRange.start;
      
      storage.createTask({
        userId,
        projectId: campaign.projectId ?? undefined,
        campaignId: campaign.id,
        title: t.title,
        description: t.description,
        type: t.type || 'planning',
        category: t.category || 'planning',
        priority: t.priority || 2,
        estimatedDuration: t.estimatedDuration || 30,
        taskEnergyType: t.taskEnergyType || 'execution',
        source: 'campaign',
        scheduledDate: dateToStr(clampedDate),
        completed: false,
      });
      tasksCreated++;
    });
  }

  // ── 3. Create content pieces in Content Calendar ──────────────────
  let contentCreated = 0;
  for (const piece of contentPlan) {
    // Parse "Week N" or "Month N" from piece.week
    let weekNum = 1;
    const wMatch = piece.week.match(/[Ww]eek\s*(\d+)/);
    const mMatch = piece.week.match(/[Mm]onth\s*(\d+)/);
    if (wMatch) weekNum = parseInt(wMatch[1]);
    else if (mMatch) weekNum = (parseInt(mMatch[1]) - 1) * 4 + 1;

    // Wednesday of that week at 10:00 AM
    const pieceDate = addDays(startDate, (weekNum - 1) * 7 + 2); // +2 = Wednesday
    pieceDate.setHours(10, 0, 0, 0);

    await storage.createContent({
      userId,
      projectId: campaign.projectId ?? undefined,
      title: piece.angle,           // The angle IS the content title/idea
      body: piece.copyDirections,   // Copy directions as the body/brief
      platform: piece.platform,
      contentType: mapFormatToContentType(piece.format), // helper below
      pillar: piece.pillar,
      goal: piece.goal,
      status: 'draft',
      contentStatus: 'idea',        // Start as "Idea" in the pipeline
      scheduledFor: pieceDate,
    });
    contentCreated++;
  }

  // ── 4. Update campaign ────────────────────────────────────────────
  const updated = await storage.updateCampaign(id, userId, {
    tasksGenerated: true,
    status: 'active',
    startDate: dateToStr(startDate),
  });

  res.json({ campaign: updated, tasksCreated, contentCreated });
});
Add this helper function before the route handler (or at the top of the routes file utilities section):
tsfunction mapFormatToContentType(format: string): string {
  const f = format.toLowerCase();
  if (f.includes('email') || f.includes('newsletter')) return 'email';
  if (f.includes('article') || f.includes('blog')) return 'article';
  if (f.includes('story') || f.includes('reel') || f.includes('video')) return 'story';
  return 'post'; // default for carousel, text post, etc.
}
