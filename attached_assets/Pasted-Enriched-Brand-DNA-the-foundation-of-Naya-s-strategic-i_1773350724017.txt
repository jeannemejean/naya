Enriched Brand DNA: the foundation of Naya's strategic intelligence
Context and philosophy
Naya cannot function as a strategic system if it doesn't truly understand the user's business. The current onboarding captures useful signals (business type, audience, tone, platform) but is missing the dimensions that allow Naya to generate real communication strategy: concrete offers, precise differentiated positioning, editorial direction, and active business priorities. This brief enriches Brand DNA without breaking anything that already works.

Part 1 — Schema: new Brand DNA fields
In shared/schema.ts, add the following fields to the brand_dna table:
ts// Offers and value
offers: text("offers"),
// Free-form description of offers: "Studio In Situ (field immersion), In Linea (90-day strategic accompaniment), In Scena (campaign production)"

priceRange: text("price_range"),
// budget | mid-market | premium | luxury | varies

clientJourney: text("client_journey"),
// How a client arrives and progresses: discovery → call → proposal → delivery → retention

// Positioning and differentiation
brandVoiceKeywords: text("brand_voice_keywords").array(),
// 3-5 words defining the brand voice: ["incisive", "elegant", "strategic", "unfiltered"]

brandVoiceAntiKeywords: text("brand_voice_anti_keywords").array(),
// What the brand is NOT: ["generic", "corporate", "overly safe"]

editorialTerritory: text("editorial_territory"),
// Editorial territory in one sentence: "The intersection of strategy and aesthetics for fashion/beauty brands"

competitorLandscape: text("competitor_landscape"),
// Who does something similar and how this brand stands apart

// Creative direction
visualIdentityNotes: text("visual_identity_notes"),
// Free notes on visual identity: colors, references, mood

contentPillarsDetailed: jsonb("content_pillars_detailed"),
// [{ pillar: "Expert Perspective", description: "...", formats: ["LinkedIn", "newsletter"], frequency: "weekly" }]

referenceBrands: text("reference_brands").array(),
// Admired brands / inspirational references

// Active business context
activeBusinessPriority: text("active_business_priority"),
// What the user is focused on RIGHT NOW: "Launch In Scena offer, land 2 clients by June"

currentBusinessStage: text("current_business_stage"),
// launch | validation | growth | consolidation | pivot | scaling

revenueTarget: text("revenue_target"),
// Revenue objective: "150k€ this year", "3 premium clients per quarter"

keyMilestones: jsonb("key_milestones"),
// [{ title: "First In Scena client signed", targetDate: "2026-06-30", status: "active" }]

// Operational realities
teamStructure: text("team_structure"),
// solo | solo-with-freelancers | small-team | growing-team

operationalConstraints: text("operational_constraints"),
// Realities that constrain: "Single creative director, occasional freelancers, no sales staff"

geographicFocus: text("geographic_focus"),
// local | national | european | international | remote-only

languageStrategy: text("language_strategy"),
// Communication languages: "Primarily French, occasionally English for strategic documents"

// Contextual intelligence (filled by Naya, not the user)
nayaIntelligenceSummary: text("naya_intelligence_summary"),
// Strategic synthesis generated by Naya after analyzing the complete profile

lastStrategyRefreshAt: timestamp("last_strategy_refresh_at"),
// When Naya last regenerated its strategic analysis
```

Run `npm run db:push` after adding these fields.

### Part 2 — Settings page: enriched Brand DNA, always accessible and editable

The Settings page (`client/src/pages/settings.tsx`) must become the place where users can view, complete, and edit their Brand DNA at any time — not only during onboarding.

Restructure the Brand DNA section in Settings into 5 tabs:

**Tab 1 — Identity**
- Business name, website, LinkedIn, Instagram (already exists)
- Business type, model, current stage
- Editorial territory (free textarea)
- Brand voice keywords (editable chips, max 6)
- Anti-keywords / what the brand is NOT (editable chips, max 4)
- Visual identity notes (textarea)
- Reference brands (chips, max 5)

**Tab 2 — Offers & Market**
- Offers description (free textarea, placeholder: "Describe your offers as you would to a prospect")
- Price range (select)
- Primary audience (existing select)
- Core audience pain point (existing select)
- Audience aspiration (existing select)
- Client journey (textarea: "How does a prospect become a client?")
- Competitor landscape (textarea: "Who does something similar? How do you stand apart?")

**Tab 3 — Content & Platforms**
- Priority platform (existing select)
- Current presence (existing select)
- Content bandwidth (existing select)
- Detailed content pillars: interface to create/edit up to 5 pillars, each with: name, short description, associated formats (multi-select: LinkedIn / Instagram / Newsletter / Blog / Video), desired frequency
- Communication style (existing select)

**Tab 4 — Active Priorities**
- Active business priority (textarea: "What are you working on right now? What is your focus for the next 3 months?")
- Revenue target (free text input)
- Team structure (select)
- Operational constraints (textarea)
- Geographic focus (select)
- Key milestones: interface to create up to 5 milestones, each with: title, target date, status (active / reached / paused)

**Tab 5 — Naya Intelligence**
- Read-only display of `nayaIntelligenceSummary` (AI-generated)
- "Refresh analysis" button that calls the endpoint described below
- Last updated date

### Part 4 — Endpoint: Naya intelligence generation

Create route `POST /api/brand-dna/refresh-intelligence` in `server/routes.ts`.

This route:
1. Loads the user's complete Brand DNA
2. Calls OpenAI with the following system prompt:
```
You are Naya, an AI business strategist. You have just received a complete brand profile.
Analyze it and produce a concise strategic intelligence summary (200-300 words) covering:
1. The brand's core strategic position in one sentence
2. The 3 most powerful communication angles available to this brand right now
3. The 2-3 content formats most likely to work given their bandwidth and platform
4. The main growth lever to prioritize in the next 90 days
5. One blind spot or risk to watch

Write as a sharp strategic advisor, not a generic AI. Be specific to this brand, not generic.
Return plain text, no headers, no bullet points — flowing strategic prose.

Saves the result to nayaIntelligenceSummary and lastStrategyRefreshAt
Returns the summary

This route is called automatically on first onboarding submission if enough fields are populated (businessType + offers + uniquePositioning + platformPriority all non-empty).
It is also called manually via the Refresh button in the Intelligence tab in Settings.

After this step, Naya genuinely knows the business: its offers, voice, differentiating positioning, active priorities, and key milestones. The nayaIntelligenceSummary becomes the base context for all future strategic generation. Without this foundation, the Strategy page cannot exist.
