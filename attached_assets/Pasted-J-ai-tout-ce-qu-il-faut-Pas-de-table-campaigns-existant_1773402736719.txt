Step 9 — Campaign Engine
What this step does
The Campaign Engine is a new page at /campaigns that lets the user define a campaign objective, have Naya generate a strategic message, key assets, and a set of ready-to-execute tasks — all linked to a selected project. It replaces the need to manually think "what do I actually do this week to move this goal forward?" The output is: a campaign record with a name, objective, core message, and a task list injected directly into the Planning page.
Task 1 — Schema: add campaigns table
In shared/schema.ts, add the following table after milestoneTriggers:
ts
export const campaigns = pgTable("campaigns", {
  id: serial("id").primaryKey(),
  userId: varchar("user_id").notNull().references(() => users.id),
  projectId: integer("project_id").references(() => projects.id),
  name: text("name").notNull(),
  objective: text("objective").notNull(),         // user's raw goal input
  coreMessage: text("core_message"),              // AI-generated key message
  targetAudience: text("target_audience"),        // AI-derived or user-refined
  duration: text("duration").default("1_week"),   // 1_week | 2_weeks | 1_month
  status: text("status").default("draft"),        // draft | active | completed | paused
  tasksGenerated: boolean("tasks_generated").default(false),
  insights: jsonb("insights"),                    // AI reasoning array
  createdAt: timestamp("created_at").defaultNow(),
  updatedAt: timestamp("updated_at").defaultNow(),
});

export const campaignRelations = relations(campaigns, ({ one }) => ({
  project: one(projects, { fields: [campaigns.projectId], references: [projects.id] }),
}));

export const insertCampaignSchema = createInsertSchema(campaigns).omit({ id: true, createdAt: true, updatedAt: true });
export type Campaign = typeof campaigns.$inferSelect;
export type InsertCampaign = z.infer<typeof insertCampaignSchema>;
