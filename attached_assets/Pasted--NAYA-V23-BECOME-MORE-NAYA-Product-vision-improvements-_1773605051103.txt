# NAYA V23 — BECOME MORE NAYA
## Product vision improvements to align with the real Naya identity

---

## WHAT NAYA IS (the north star for every change below)

Naya is not a task manager with AI features.
Naya is **an AI Operating System that knows how you think, work, and build** — and organizes your entire business life around that knowledge.

The key promise: you open Naya every morning and instantly know what to do, why, and in what order — because Naya already knows your energy, your psychology, your bottlenecks, and your goals.

The differentiator: **Naya doesn't just generate tasks. Naya knows YOU.**

Every word, screen, and feature should reinforce this. Right now, several screens feel like generic SaaS. Here's what needs to change.

Replace the briefPrompt variable with this enhanced version:

## CHANGE 3 — DASHBOARD: Show the Operating Profile visibly

**File:** `client/src/pages/dashboard.tsx`

**Problem:** The user's behavioral profile (persona, energy rhythm, avoidance triggers) is collected in onboarding but is almost invisible in the dashboard. The persona chip is tucked away. Users don't feel "known" by Naya.

**Add a compact "Your Profile" card** to the dashboard, visible only when the user has a persona detected. It should appear below the DailyBrief, above the task list.

The card should show:
- Persona name + icon (Builder ⚡, Strategist 🧭, Creative 🎨, Analytical 📊)
- Energy rhythm (e.g., "Morning performer")
- One active avoidance trigger if present (e.g., "Watch for: perfectionism")
- A link to Settings to update

Example design (add this as a new component `OperatingProfileChip` in dashboard.tsx):

```tsx
function OperatingProfileChip() {
  const { data: personaResult } = useQuery<PersonaAnalysisResult | null>({
    queryKey: ['/api/persona/my-persona'],
  });
  const { data: opProfile } = useQuery<UserOperatingProfile | null>({
    queryKey: ['/api/user/operating-profile'],
  });

  const personaName = (personaResult?.analysisResult as any)?.personaName as string | undefined;
  if (!personaName) return null;

  const PERSONA_ICONS: Record<string, string> = {
    Strategist: "🧭", Builder: "⚡", Creative: "🎨", Analytical: "📊",
  };

  const avoidance = (opProfile?.avoidanceTriggers as string[] | undefined)?.[0];
  const rhythm = opProfile?.energyRhythm;

  return (
    <div className="flex items-center gap-3 px-3 py-2 bg-slate-50 dark:bg-gray-800/50 rounded-lg border border-slate-200 dark:border-gray-700 mb-3">
      <span className="text-base">{PERSONA_ICONS[personaName] || "🧠"}</span>
      <div className="flex-1 min-w-0">
        <p className="text-xs font-semibold text-slate-800 dark:text-white">{personaName}</p>
        {rhythm && <p className="text-[10px] text-slate-400 dark:text-gray-500 truncate">{rhythm}</p>}
      </div>
      {avoidance && (
        <span className="text-[10px] text-amber-600 dark:text-amber-400 bg-amber-50 dark:bg-amber-900/20 px-2 py-0.5 rounded-full border border-amber-200 dark:border-amber-800/40 whitespace-nowrap">
          Watch: {avoidance}
        </span>
      )}
    </div>
  );
}
```

Place `<OperatingProfileChip />` inside the main dashboard layout, just after `<DailyBrief />`.
